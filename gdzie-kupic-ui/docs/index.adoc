= My Document

== Actors

* User: The person creating the post.
* System: The backend that processes the post and manages the workflow.
* Merchant: The person or entity that may be associated with the post, depending on the category selected. Can respond to the post with offers or information.
* Merchant Agent: An automated agent that interacts with the post on behalf of the merchant, providing responses or offers based on the content of the post.

== Post Creation Process

[plantuml]
----
@startuml
* User clicks "Post" button
* User selects Category
* User writes content
* User clicks "Submit"
* System validates input
* System saves post to database
* System schedules agents to process post
* System sends confirmation back to user
* User can see post on the website
* User can see how many merchants viewed the post
* User can see how many merchants responded to the post
@enduml
----

== Post Processing by Merchant

[plantuml]
----
@startuml
* Merchant receives notification of new post in category of interest
* Merchant reviews post content
if (Merchant decides to respond?) then (yes)
  * Merchant writes response
  * Merchant submits response
  * System validates response
  * System saves response to database
  * System notifies user of response
else (no)
  * Merchant ignores post
* System notifies user of response
@enduml
----

== Merchant Registration Process

[plantuml]
----
@startuml
* Merchant visits registration page
* Merchant fills out registration form
* Merchant submits registration
* System validates registration information
* System creates merchant account
* System sends confirmation email to merchant
* Merchant can log in to the system
@enduml
----

== User Interactions with Post

- User can view post responses from merchants.
- User can message merchants in post responses.
- User can close the post when they have found a satisfactory response.
- User can rate merchant responses.
- User can close the post anytime.

== User Authentication

- Unregistered users receive a long-lived token that allows them to create posts and interact with the system without needing to log in. User can choose to register at any time, which will link their existing posts and interactions to their new account. Token expires after *1 year*. Lost token cannot be recovered - only registration can save the user's data.

- Users are able to register with email and password or via OAuth providers (Google, Facebook, etc.). Registered users can log in to access their posts and interactions across devices.
